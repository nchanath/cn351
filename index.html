LAB 10 -- วันที่ 21 มีนาคม พ.ศ. 2565

Injection flaws

ในแลบนี้นักศึกษาจะใช้ web security dojo VM

  1. ทำแบบฝึกหัด WebGoat เรื่อง Injection Flaws โดยให้ทำแลบในหัวข้อต่อไปนี้
  • Command Injection
  • Numeric SQL Injection
  • Log Spoofing
  • XPATH Injection
  • String SQL Injection
  • LAB: SQL Injection Stages 1 and 3
  • Blind Numeric SQL Injection
  • Blind String SQL Injection
  1. ให้ทำแบบฝึกหัด dvwa ในหัวข้อ SQL Injection เพียงหัวข้อเดียว โดยให้
  • แสดงผลลัพธ์บน browser และ
  • แสดงการส่ง SQL query ที่ใช้ในการโจมตีในข้อนี้ ผ่าน mysql client ที่ terminal และแสดงผลลัพธ์ที่ได้ว่ามีเนื้อหาที่ตรงกันกับผลลัพธ์ที่แสดงบน browser

ในการทำแลบทั้งสองส่วนให้ใช้

  • burp suite สำหรับดักจับข้อมูลระหว่าง browser และ server และ
  • firefox ในการสำรวจ source code

เท่านั้น ห้ามใช้เครื่องมืออำนวยความสะดวกอื่น ๆ เช่น Tamper Data, Live HTTP headers ฯลฯ

รายงานแลบ

ให้เขียนสรุปสั้น ๆ เพื่ออธิบายในแต่ละข้อถึง

  • สาเหตุที่ทำให้เกิดช่องโหว่ และ
  • แนวคิดในการโจมตีของนักศึกษา (เขียนโดยละเอียดพอประมาณ) และ
  • SQL query (หรือ queries) ที่นักศึกษาคิดว่าถูกส่งไปยังฐานข้อมูลเมื่อนักศึกษาทำการโจมตี และคำอธิบายผลลัพธ์ที่ได้ ว่าเกิดจาก SQL query แต่ละค่าที่ส่งไปได้เพราะอะไร อย่างไร และ
  • แนวคิดในการปิดหรือรับมือกับช่องโหว่ (เขียนโดยละเอียดพอประมาณ)

LAB 9 -- วันที่ 14 มีนาคม พ.ศ. 2565

Access control flaws

ในแลบนี้นักศึกษาจะใช้ web security dojo VM และทำแบบฝึกหัด WebGoat เรื่อง Access Control Flaws โดยให้ข้าม Stage 2 และ 4

ขั้นตอนการทำแลบ

  1. Boot เครื่องเสมือนชื่อ web security dojo
  2. Start WebGoat
  3. ทำแลบ Access Control Flaws โดยให้ใช้
    • burp suite สำหรับดักจับข้อมูลระหว่าง browser และ server และ
    • firefox ในการสำรวจ source code
    เท่านั้น ห้ามใช้เครื่องมืออำนวยความสะดวกอื่น ๆ เช่น Tamper Data, Live HTTP headers ฯลฯ
  4. ส่งรายงานแลบที่มีสองส่วนต่อไปนี้
    • สำหรับหัวข้อ Using an access control matrix ให้เขียนตารางแสดง access control matrix โดยให้แยกเป็น 2 ตาราง ตารางที่ 1 แสดงความสัมพันธ์ระหว่าง usernames และ user roles ตารางที่ 2 แสดงความสัมพันธ์ระหว่าง user roles และ resources
    • สำหรับหัวข้อที่เหลือ ให้เขียนสรุปสั้น ๆ เพื่ออธิบายในแต่ละข้อถึง
      • สาเหตุที่ทำให้เกิดช่องโหว่ และ
      • แนวคิดในการโจมตีของนักศึกษา (เขียนโดยละเอียดพอประมาณ) และ
      • แนวคิดในการปิดหรือรับมือกับช่องโหว่ (เขียนโดยละเอียดพอประมาณ)

    ห้ามใช้เครื่องมืออำนวยความสะดวกอื่น ๆ เช่น Tamper Data, Live HTTP headers ฯลฯ
  4. เขียนข้อสรุปสั้น ๆ เพื่ออธิบายในแต่ละข้อถึง

LAB 7 -- วันที่ 21 กุมภาพันธ์ พ.ศ. 2565

Web Authentication Vulnerabilities

ในแลบนี้นักศึกษาจะใช้ web security dojo VM และทำแบบฝึกหัด WebGoat เรื่อง Authentication Flaws โดยให้ข้ามหัวข้อแรกคือเรื่อง Password Strength

ขั้นตอนการทำแลบ

  1. Boot เครื่องเสมือนชื่อ dojo
  2. ใช้ burp suite และ firefox
  3. ตั้งค่าให้ burp ดักจับ connection ของ firefox
  4. Start WebGoat
  5. ทำแลบ WebGoat ในส่วน Authentication Flaws โดยให้ใช้
    1. Burp (ใช้สำหรับดักจับข้อมูลระหว่าง browser และ server) และ
    2. firefox dev tools (ใช้ในการสำรวจ source code)
    เท่านั้น ห้ามใช้เครื่องมืออำนวยความสะดวกอื่น ๆ เช่น Tamper Data, เว็บไซต์ที่ช่วยในการ encode/decode base64, Live HTTP headers ฯลฯ
  6. เขียนข้อสรุปสั้น ๆ สำหรับแบบฝึกหัดแต่ละข้อให้ครบทั้ง 3 ข้อ (Forgot Password, Multi Level Login 1, Multi Level Login 2) โดยในแต่ละข้อให้อธิบายคำตอบในสองประเด็นต่อไปนี้
    1. สาเหตุที่ทำให้เกิดช่องโหว่คืออะไร
    2. วิธีการปิดช่องโหว่หรือรับมือกับช่องโหว่นั้นมีอะไรบ้าง

    7. ข้อมูลที่อาจเป็นประโยชน์

    • นักศึกษาสามารถศึกษาการใช้งานคำสั่ง openssl ได้โดยการพิมพ์ man openssl ที่ command prompt

    สิ่งที่ต้องส่ง

    • วิดีโอแสดงการทำแลบ กำหนดส่ง 23.59 น. 24 กุมภาพันธ์ 2565
    • คำตอบในข้อสุดท้าย กำหนดส่ง 23.59 น. 24 กุมภาพันธ์ 2565

LAB 6 -- วันที่ 14 กุมภาพันธ์ พ.ศ. 2565

Bypassing Client-Side Controls

ในแลบนี้นักศึกษาจะใช้ web security dojo VM และทำแบบฝึกหัด WebGoat เรื่อง Parameter Tampering

วัตถุประสงค์

  • เพื่อศึกษาการแก้ไขข้อมูลทั้งในส่วนของ browser request และ server response เพื่อให้เห็นว่าการตรวจสอบข้อมูลที่ user กรอกด้วยกลไกต่าง ๆ บน client นั้น เป็นสิ่งที่ไม่มีประสิทธิภาพเพียงพอในการป้องกันการโจมตีในรูปแบบต่าง ๆ
  • เพื่อแสดงให้เห็นว่า hidden fields นั้น จริง ๆ แล้ว ถูกซ่อนไว้บนหน้าจอของ web browser เท่านั้น user บนเครื่อง client ยังสามารถเข้าถึงข้อมูลส่วนนี้ได้โดยง่าย
  • เพื่อให้เห็นความสำคัญของการตรวจสอบความถูกต้องของข้อมูลที่ผู้ใช้กรอก (โดยผู้ตรวจสอบต้องเป็นฝั่ง server)
  • เพื่อศึกษาการใช้งาน burp suite
  • เพื่อศึกษาการใช้งาน firefox developer tools

ขั้นตอนการทำแลบ

  1. Boot เครื่องเสมือนชื่อ web security dojo
  2. Start โปรแกรม firefox
  3. Start โปรแกรม burp suite
  4. ตรวจสอบว่า burp ดักจับ TCP connection ที่ port ใด แล้วตั้งค่าให้ burp ดักจับ connection ของ firefox
  5. start webgoat ในเครื่อง
  6. ทำแลบ Webgoat Parameter Tampering
    1. ทำแลบโดยใช้ burp suite
    2. ทำแลบโดยใช้ firefox developer tools
  7. การโจมตีในข้อ 6.1 มี**แนวคิด**อย่างไร จงอธิบาย
  8. การโจมตีในข้อ 6.2 มี**แนวคิด**อย่างไร จงอธิบาย
  9. การโจมตีในข้อ 6.1 และ 6.2 มี**แนวคิด**ต่างกันอย่างไร จงอธิบาย
  10. จงหาคำตอบว่า Javascript code ในการ validate รูปแบบของข้อมูลนั้น มีลักษณะอย่างไร อยู่ที่ไหน ให้นักศึกษาเขียนคำบรรยายสรุปการทำงานของโค้ดในส่วนนี้

ข้อมูลที่อาจเป็นประโยชน์

  • ข้อมูลพื้นฐานเกี่ยวกับ Javascript regular expression object: http://www.w3schools.com/jsref/jsref_obj_regexp.asp

สิ่งที่ต้องส่ง

  • วิดีโอแสดงการทำแลบ กำหนดส่ง 23.59 น. 17 กุมภาพันธ์ 2565
  • คำตอบในข้อ 7-10 กำหนดส่ง 23.59 น. 17 กุมภาพันธ์ 2565

LAB 5 -- วันที่ 7 กุมภาพันธ์ พ.ศ. 2565

เรียนรู้พื้นฐานในการใช้งานโปรแกรม Burp Suite

ในแลบนี้ ให้นักศึกษาใช้ dojo VM แล้วปฏิบัติตามวิดีโอ Burpsuite Basics (FREE Community Edition) แล้วอัดวิดีโออธิบายสิ่งที่นักศึกษาทำ ตามลำดับที่ปรากฎในวิดีโอต้นแบบ

เนื่องจากใน dojo VM มีการติดตั้ง DVWA และ Burp Suite Community Edition มาเรียบร้อยแล้ว นักศึกษาจึงสามารถเริ่มปฏิบัติตามวิดึโอต้นแบบได้ตั้งแต่นาทีที่ 7:24 เป็นต้นไป

ข้อควรตระหนัก

  1. เนื่องจาก Burp Suite Community Edition ที่ได้รับการติดตั้งใน dojo VM และในเครื่องของผู้จัดทำวิดีโอต้นแบบเป็นคนละเวอร์ชั่นกัน เมนูและการทำงานบางอย่างจะไม่เหมือนกันเสียทีเดียว ให้นักศึกษาพยายามทำความเข้าใจและหาทางออกด้วยตนเองและสิ่งรอบตัว ทักษะนี้เป็นส่วนหนึ่งที่นักศึกษาต้องเรียนรู้และต้องทำการฝึกฝน

  2. ในวิดีโอมีการแสดงการโจมตีชนิด SQL Injection (sqli) ให้นักศึกษาทำตามผู้จัดทำวิดีโอ โดยยังไม่ต้องทำความเข้าใจทฤษฎีพื้นฐานเรื่องนี้ เนื่องจากจะมีการศึกษาโดยละเอียดในแลบอื่นในวิชานี้ต่อไป

  3. หากนักศึกษาสนในการใช้ Burp Suite ทำ spidering (หรือ crawling สำหรับ Burp version ปัจจุบัน) และ scanning (หรือ auditing) นักศึกษาสามารถลอง trial version ของ Burp Suite Professional Edition ได้ แต่ต้องทำการทดลองภายใน 30 วัน นักศึกษาไม่จำเป็นต้องทำในส่วนนี้สำหรับแลบนี้

สิ่งที่นักศึกษาควรเรียนรู้ในแลบนี้

  • การใช้ Burp Suite ในขั้นพื้นฐาน กล่าวคือการใช้ Tabs ต่อไปนี้คือ Target, Proxy, Repeater, Decoder, Comparer และ User options

สิ่งที่ต้องส่ง

วิดีโอ โดยให้นักศึกษาจัดระเบียบให้ครบถ้วนและมีลำดับการแสดงให้เหมือนกับที่ผู้จัดทำวิดีโอได้ทำไว้ ลองดู Handout 3 เป็นตัวอย่างได้ แต่เนื่องจากแลบนี้ค่อนข้างไม่ซับซ้อน ให้นักศึกษาเขียน script ด้วยตนเอง

LAB 4 -- วันที่ 31 มกราคม พ.ศ. 2565

เรียนรู้พื้นฐานที่เกี่ยวข้องกับ HTTP

ให้นักศึกษาปฏิบัติตามวิดีโอ HTTP Crash Course & Exploration แล้วอัดวิดีโออธิบายสิ่งที่นักศึกษาทำ ตามลำดับที่ปรากฎในวิดีโอต้นแบบ

นักศึกษาสามารถคัดลอก code ต่อไปนี้ไปใช้เป็นต้นแบบได้

สิ่งที่นักศึกษาควรเรียนรู้ในแลบนี้

  • การใช้งาน web browser developer tools เช่น Firefox หรือ Chrome
  • การติดตั้งและตั้งค่า node.js และ express.js สำหรับทำการทดลองส่ง request และรับ response และตรวจสอบค่า HTTP messages ที่ส่งไปให้และได้รับจากเซิร์ฟเวอร์

สิ่งที่ต้องศึกษาสำหรับแลบนี้

LAB 3 -- วันที่ 24 มกราคม พ.ศ. 2565

ส่วนที่ 1: ศึกษาความแตกต่างระหว่างการแลกเปลี่ยนข้อมูลเมื่อใช้ method GET และ method POST

  1. สร้างไฟล์ form-w-get.html เพื่อแสดงแบบฟอร์มที่มีช่องรับอินพุตเป็น text เพียงหนึ่งช่อง ให้อินพุตนี้มีชื่อว่า form-input และให้ใช้ method เป็น GET
  2. สร้างไฟล์ form-w-post.html เพื่อแสดงแบบฟอร์มที่มีช่องรับอินพุตเป็น text เพียงหนึ่งช่อง ให้อินพุตนี้มีชื่อว่า form-input และให้ใช้ method เป็น POST
  3. สร้างไฟล์ get-formhandler.php และ post-formhandler.php สำหรับประมวลผลข้อมูลที่ได้รับจากการกรอกแบบฟอร์มแบบ GET และแบบ POST ตามลำดับ โดยให้แสดงผลลัพธ์เป็น form-input = x โดย x คือ ค่าที่ได้รับจากการกรอกแบบฟอร์มของผู้ใช้
  4. นำไฟล์ form-w-get.html form-w-post.html get-formhandler.php และไฟล์ post-formhandler.php ไปวางไว้ในที่ที่เหมาะสม เพื่อให้สามารถเข้าถึงได้โดยผ่าน URL http://targets.local/lab2/….
  5. ทดสอบโดยใช้ burp suite และ developer tools ใน firefox
  6. พิจารณาว่าการใช้ GET และ POST นั้นมีความแตกต่างกันอย่างไร แล้วเขียนบรรยายตามความเข้าใจของตนลงในบันทึกแลบ

ส่วนที่ 2: ทดสอบการรับส่ง hidden variables

  1. ใน directory ชื่อ lab2 สร้างไฟล์ hidden-var-form.html เพื่อแสดงแบบฟอร์มที่มีช่องรับอินพุตเป็น text เพียงหนึ่งช่อง ให้อินพุตนี้มีชื่อว่า normal-var และมีการส่ง hidden variable หนึ่งตัวชื่อ hidden-var
  2. สร้างไฟล์ hidden-var-formhandler.php เพื่อประมวลผลข้อมูลที่ได้รับจากการกรอกแบบฟอร์ม โดยให้แสดงผลลัพธ์เป็น normal-var = x and hidden-var = y โดย x คือ ค่าที่ได้รับจากการกรอกแบบฟอร์มของผู้ใช้ และ y คือ ค่าของ hidden variable
  3. นำไฟล์ hidden-var-form.html และไฟล์ hidden-var-formhandler.php ไปวางไว้ในที่ที่เหมาะสม เพื่อให้สามารถเข้าถึงได้โดยผ่าน URL http://targets.local/lab2/….
  4. ทดสอบโดยใช้ burp suite และ developer tools ใน firefox
  5. อธิบายว่า hidden variables คืออะไร การรับ-ส่ง hidden variable มีลักษณะอย่างไร

ส่วนที่ 3: ทดสอบการรับส่ง cookies

  1. สร้างไฟล์ชื่อ setcookie.php โดยให้มีการทำงานคือ ให้สั่ง server ให้ส่ง cookie ที่มี name เป็น remember และ value เป็น ชื่อของนักศึกษา
  2. สร้างไฟล์ชื่อ usecookie.php โดยให้มีการทำงานคือ ให้สั่ง server ให้แสดงค่า cookie ที่ได้รับมาจาก browser
  3. ใช้ burp suite และ firefox quantum ในการสำรวจเนื้อหาของข้อมูลที่มีการแลกเปลี่ยนกันสำหรับทั้งสอง requests ข้างต้น
  4. อธิบายว่า cookies คืออะไร การรับ-ส่ง cookies มีลักษณะอย่างไร
