LAB 11 -- วันที่ 28 มีนาคม พ.ศ. 2565

Cross-site scripting (XSS)

แลบวันนี้มีเนื้อหาเกี่ยวกับ Cross site scripting (XSS) vulnerabilities ให้นักศึกษาทำแลบดังต่อไปนี้

  1. Introduction to JavaScript ที่ W3schools เรื่อง JavaScript tutorial http://www.w3schools.com/js/default.asp โดยให้ทำ 4 หัวข้อแรก (home, introduction, where to, output)

  2. ในเครื่อง Dojo VM

    • สร้างแบบฟอร์ม hello-whoever-form.html และสร้างโปรแกรมชื่อ hello-whoever.php ที่สามารถแสดงคำว่า Hello xx! ในเว็บบราวเซอร์ โดย xx คือ คำที่ผู้ใช้กรอกลงไปในแบบฟอร์ม และ method ในการเรียกโปรแกรมเป็น POST จากนั้นให้นำไฟล์ทั้งสองไปวางไว้ในที่ที่เหมาะสม เพื่อให้สามารถเข้าถึงได้โดยผ่าน URL http://targets.local/lab11/hello-whoever-form.html
    • เพิ่มการตั้งค่า cookie ใน hello-whoever.php โดยนักศึกษาสามารถตั้งค่า cookie ให้เป็นค่าใดก็ได้ที่ต้องการ
    • แสดงให้เห็นว่าการทำงานของสองไฟล์นี้ ก่อให้เกิดช่องโหว่แบบ cross-site scripting โดยการแสดงให้เห็นว่า ผู้ร้ายสามารถใช้ javascript ทำการ pop up หน้าต่างที่แสดงค่า cookie ของผู้ใช้ได้ เมื่อผู้ใช้ถูกหลอกให้พิมพ์ค่าอินพุตบางอย่างลงในแบบฟอร์ม (หรือถูกหลอกให้ submit แบบฟอร์มที่มีค่าอินพุตนั้นโดยไม่รู้ตัว)

  3. ในเครื่อง Dojo VM ทำแลบ WebGoat ในหัวข้อต่อไปนี้

    • Phishing with XSS
    • Stored XSS attacks
    • LAB: Cross Site Scripting: stages 1, 3, 5
    • Reflected XSS attacks

รายงานแลบ

สำหรับแต่ละหัวข้อในส่วน WebGoat ให้เขียนสรุปสั้น ๆ เพื่ออธิบายในแต่ละข้อถึง

  • สาเหตุที่ทำให้เกิดช่องโหว่ และ
  • แนวคิดในการโจมตีของนักศึกษา (เขียนโดยละเอียดพอประมาณ) และ
  • แนวคิดในการปิดหรือรับมือกับช่องโหว่ (เขียนโดยละเอียดพอประมาณ)
